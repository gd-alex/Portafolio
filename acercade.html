Acerca de

Para desarrollar este portafolio, lo primero que hice es crear un Notion para anotar ahí toda la información referida al UX. User Experience o "UX" es la parte de la planificación e investigación para brindar una mejor experiencia al usuario que visite nuestra página. No soy un experto en este campo, pero sé que es importante debido a que nos brinda una dirección sobre la cual encaminarnos a la hora de realizar una página.

Hice un análisis del anterior portafolio web, investigué sobre otros portafolios; establecí un público objetivo, el objetivo de la página, una estrategia para cumplir ese objetivo; hice un mapa de sitio, diagrame la página (wireframes), y por último definí cuál sería la información que le correspondería a cada sección/página.

Luego, procedí a la parte visual de la página (UI). Comencé por crear el Design System, el cuál esta compuesto por colores, tipografía, espaciado y muchos componentes reutilizables como botones, cards, etc. Me base en un estilo moderno y "liquid", que fuera distintivo y memorable. Mi idea era crear una experiencia que perdure en la mente del usuario, algo atípico, que muestre lo que se puede lograr en una página web.

El por que se crea el Design System, es la misma razón que la del UX, es la base que establece el camino que tomará la página, pero en este caso, es el camino visual.

Una vez terminado el Design System, continué con el prototipado. Esta parte es la que más demora en el campo visual. Tomó aproximadamente 5-7 días y fue realizado en Figma.

Para esto utilicé el sistema de columnas de Google, que es bastante conocido en el medio. También establecí jerarquías, posicioné CTA's, cree la interactividad, las animaciones, adapté a tablet y celular e hice pruebas para el dark mode.

Ahora llegamos a la parte del código. Para esto utilicé Astro + CSS + JS + Tailwind. De todos estos, el único que nunca había utilizado era Tailwind, que decidí implementarlo para probar que tan útil era. Mi recomendación, si estás empezando en el código y quieres tener un portafolio web, es que vayas por los básicos que son HTML, CSS y JavaScript (aunque este último no es tán básico), estos te permitirán tener una web sólida y funcional, y sirven perfectamente para proyectos no tan grandes.

Yo utilizo Astro y Tailwind (que son frameworks) debido a que permiten un trabajo más profesional, aunque claramente se vuelve más complicado el proyecto al usar estos frameworks. Lo positivo es que al usarlos puedo realizar todos los cambios que desee y no morir en el intento o por viejo.

Dentro de Astro creo componentes que me permiten realizar muchos cambios desde un solo archivo, Tailwind me agiliza el trabajo con estilos en linea, y JavaScript permite que el sitio funcione de manera mucho más profesional.

Una vez terminado, subo a GithubPages los archivos compilados (esto es por que uso Astro, pero si solo utilizas HTML + CSS + JS no deberías preocuparte por esto, basta con que subas la carpeta donde está tu código). Y ese es el proceso un tanto resumido, con la información esencial para que puedas realizar tu propio portafolio o saber más cosas acerca de este mismo!

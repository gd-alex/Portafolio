Proyectos

Este fue mi primer proyecto de Diseño UX/UI. Trataba de rediseñar una página web en mal estado usando Figma y los conceptos que nos iban enseñando. Como siempre, se definió un buyer persona, se investigó la página, sus problemas, etc. El resultado para ser la primera vez, fue bueno, tenía varias fallas obviamente (no era responsive, no tenía sistema de columnas, no había un design system sólido, etc.).

Aprendí bastante también, aprendí lo que era un mapa de sitio, la interactividad, los wireframes, componentes, etc. Lo disfruté bastante y me permitió conocer y abrirme a un campo que desconocía totalmente.

2do ciclo de Diseño UX/UI, esta vez tocaba crear una landing page de un producto real usando Figma para el proceso de Diseño, y HTML + CSS + Bootstrap para el Desarrollo. Está fue mi primera vez desarrollando mediante código un proyecto web, por lo que si fue díficil acostumbrarme al inicio, pero luego le agarre un gusto a usar código para mis proyectos webs.

El resultado me agradó mucho en su momento, y a comparación del anterior proyecto, hubieron bastantes mejoras con respecto al Design System y al diseño de columnas, además de implementar el responsive design y los layouts.

Este proyecto fue únicamente Figma, y no para el instituto. Lo desarrolle para practicar durante vacaciones. Y para esto, anteriormente, cree desde cero la identidad visual de la marca, que también lo incluí como proyecto y pueden verlo en este portafolio.

La idea era simple, crear un home con un mapa, que al seleccionar una ciudad, aparece una lista de los platos típicos que le corresponden. También desarrolle otras páginas como Nosotros, Talleres y Contacto. Este proyecto me tomo aproximadamente 1 semana, y aunque tuve varios problemas con el mapa, pude aprender bastante de como funciona las formas en Figma.

Por último, y el más importante, la página web de Digitales. Aquí definitivamente, aprendí el triple como mínimo de todo lo que sabía. Lo más complicado estuvo en el desarrollo (código), que tomó como 1 semana y media o 2. Siendo honesto, hubo demasiada frustración durante el proyecto, ya que implementé cosas que nunca había usado como Astro y JavaScript. Además de que me atreví a crear componentes complicados de realizar, como formularios, un cambiador de imagen (no se como se llama), y animaciones mucho más difíciles. Tuve que usar la terminal por primera vez, y en general fue una experiencia bastante buena y tediosa a la vez.
